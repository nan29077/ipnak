## 0. 한눈에 보는 결론

- **거의 모든 기능이 실제 DB(Prisma/SQLite)에 연결되어 동작**한다. 화면 → API 라우트 → Prisma → SQLite 까지 end-to-end로 배선되어 있음. mock은 "외부 유료/서드파티 연동"에 한정됨(소셜로그인·결제·이미지 스토리지·지도 SDK·AI 계측).

- **미완성(고아) 기능 1건:** `Notification`(알림) 모델은 스키마·시드(5건)까지 있으나 **앱 어디에서도 읽거나 쓰지 않음** = UI/노출 미구현.

## 2. 주요 기능별 플로우

**데이터피싱(GPS 기록):** `/map` 진입 → 현재위치 취득(`navigator.geolocation`, 거부 시 mockRoute) → "시작" → `watchPosition`으로 RoutePoint 누적·polyline·거리(Haversine) 갱신 → 일시정지/종료 → `POST /api/trips`(trip+routePoints 저장) → `/trip` 목록 반영. 지도에는 `GET /api/points`의 공개 피싱포인트도 함께 표시.

**물고기 기록 + 스마트 자:** `/catch/new` → 사진 촬영/선택(PhotoPicker) → SmartRuler에서 ①기준자 시작·끝점 탭 + 기준길이(cm) ②물고기 입·꼬리 탭 → `measuredLengthCm`·confidence 산출 → 어종/분류/채비/쇼핑태그/위치공개 입력 → `POST /api/catch`. 서버가 **FishingPoint + CatchRecord(+GearSetup) 생성**, `shareToFeed`면 FISHING_POINT 게시글까지 자동 생성.

**대회 참가:** `/tournaments` → 상세 → "참가/제출"(TournamentSubmit) → 사진 + SmartRuler 계측 → `POST /api/tournaments/[id]/entries`(status=REVIEW로 저장) → 관리자가 `/admin/reviews`에서 승인/반려 → 리더보드 반영.

**중고피싱 마켓:** 등록 `POST /api/market`(이미지·카테고리·상태) → 목록/검색 `GET /api/market` → 상세에서 찜 `POST /api/market/[id]/favorite`, 채팅 `POST /api/market/[id]/chat`(구매자별 채팅방 upsert) → 메시지 `/api/market/chats/[id]/messages` → 판매자 본인 상태변경/수정 `PATCH /api/market/[id]`(판매중→예약중→판매완료), 삭제 `DELETE`.

**예약:** `/reservations` 목록(지역·가격 필터) → 상세 → BookingForm(날짜·인원) → `POST /api/bookings`(총액=단가×인원, status=REQUESTED) → `/me`·`/admin/bookings`에서 상태 관리. **실결제 없음.**

**피드 상호작용:** 좋아요/북마크/공유/신고/숨김/댓글 각각 전용 라우트(`/api/posts/[id]/*`)로 DB 반영. 팔로우는 `/api/users/[id]/follow`(토글).

## 3. 기능별 동작 점검 (실동작 / mock / 미완성)

### ✅ 실제 동작 (DB 연결 확인됨)
인증·세션, 피드 및 모든 상호작용(좋아요·댓글·북마크·공유·신고·숨김), 게시글/물고기 기록 작성, GPS 동선 기록·세션 목록, 스마트 자 계측, 피싱포인트 지도·위치 흐림, 대회 목록·출품·심사, 예약 신청·상태관리, 중고 마켓 전체(등록·검색·찜·채팅·상태변경·삭제), 팔로우, 쇼핑태그 상품 조회, **관리자 전 메뉴(통합 액션 핸들러 + AdminLog)**. → 라우트마다 `prisma.*` 호출과 인증 가드(`requireUser`/role 체크)가 정상 배선됨.

### ⚠️ 의도된 mock (로컬/플레이스홀더 — 4장 참고)
소셜 로그인(토스트만), 이미지 업로드(미리보기 + picsum URL 저장), 예약 결제, 쇼핑 결제(외부 `buyUrl` 또는 비활성), 지도 SDK(Leaflet/OSM로 실동작하나 카카오/네이버는 스텁), AI 자동 계측(수동 측정만).

### ❌ 미완성 / 깨짐
- **`Notification`(알림): 고아 기능.** 스키마·시드(5건)만 존재, `grep` 결과 앱 코드 어디서도 읽기/쓰기 없음. 알림 생성 트리거도, 표시 UI도 없음.
- **현재 `prisma/dev.db` 파일 손상(malformed).** SQLite integrity_check 실패. 백업 `dev.db.bak-1782436520`는 정상이며 시드 데이터 완전(아래). 현재 DB로는 앱이 정상 동작하지 않으므로 재시드/복구 필요.
